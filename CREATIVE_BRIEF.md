## Design Direction Notes for Downstream Agents

### Left Panel (Context Panel) — Split Layout
The left panel shows a **simulated webpage** with highlighted/selected text and a right-click context menu showing "Send to ChatGPT" as the active item. This is the trigger state. Background: `--section-dark` (near-black with violet undertone). Show the browser context realistically — a snippet of web content (could be a documentation page, article, or code reference) with text highlighted in the teal accent color.

### Right Panel (Interactive Panel) — Split Layout
The right panel simulates the **ChatGPT interface** with the selected text pasted into the conversation. Show:
- ChatGPT-like dark interface (stay on-brand with their palette)
- The highlighted text appearing as a user message
- A simulated AI response loading/completing
- The loading state the client mentioned (spinner or streaming dots)

### Demo Screens / Feature Pages
The demo should show the extension's three states:
1. **Idle state** — Extension popup showing settings, history of sent snippets, status indicator
2. **Active state** — Text selected on page, context menu visible
3. **Result state** — ChatGPT tab with content loaded

### Typography Notes
No font override needed — Geist Sans is ideal. It is the font Linear and Vercel use, and it reads as precision developer tooling immediately. Use Geist Mono for:
- Keyboard shortcut labels (Ctrl+Shift+G)
- DOM selector strings
- Code snippets in the context menu simulation
- Any "sent X characters" or "copied from [url]" metadata

### Motion Notes
- Transitions at 120-150ms ease-out
- The "send to ChatGPT" flow can have a brief 150ms burst animation (the text visually "flying" from left to right panel)
- Loading state in the ChatGPT panel: streaming dots animation at ~80ms per dot cycle
- No theatrical page transitions — this is a utility, not a marketing site

### Vocabulary to Use in UI Labels
From the job post: "right-click context menu", "Manifest V3", "DOM manipulation", "loading states", "retry mechanism", "keyboard shortcut". Use these exact terms in the demo UI — sidebar nav items, feature labels, settings panel headers.

Suggested nav items for any sidebar or settings panel:
- Context Menu Rules
- Shortcut Settings
- Session History
- Retry Config
- Extension Stats

## Format: split-panel-demo Implementation Notes

**Left panel** (dark context surface, ~40% width):
- Shows: App name "GPT Bridge" (or similar), pulsing dot badge, feature highlights of the extension, cross-tab links, micro-CTA, attribution
- May also include: a small simulation of the selected-text context to prime the right panel demo

**Right panel** (interactive, ~60% width):
- The actual interactive demo
- Toggle between states: "Idle" | "Text Selected" | "Sent to ChatGPT"
- On "Sent to ChatGPT" state: render a ChatGPT-like interface showing the text pasted and a loading/streaming response

The `SplitShowcaseLayout` component handles the panel structure. The right panel is the demo area.
